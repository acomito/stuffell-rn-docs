SearchResults

Used on the search screen. Has gnarly logic. It is built by passing in a "keyName" which is the name of the property on the apollo response you want to show in this infinite loading list. If you have a vanilla apollo query response held in the "data" property, you would pass "data". If you renamed your apollo query e.g graphql(QUERY, { name 'customName'})() then you pass in 'customName'. We do this because we ideally want to take advantage of the refetch function on the apollo data object, to power the infinite loader. We could refactor.
Properties:
